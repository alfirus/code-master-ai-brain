# Technology Stack Preferences

## Preferred Technologies

### Frontend
- **React**: For complex UI applications
- **Next.js**: For full-stack React applications
- **Vue.js**: For lightweight applications
- **Tailwind CSS**: For styling (utility-first approach)

### Backend
- **Node.js**: For JavaScript/TypeScript backend
- **Python**: For data processing and ML
- **Go**: For high-performance services
- **PostgreSQL**: Primary database choice
- **MongoDB**: For document-based data

### DevOps & Infrastructure
- **Docker**: Containerization
- **GitHub Actions**: CI/CD
- **Vercel/Netlify**: Frontend deployment
- **AWS/Cloudflare**: Hosting services

### Tools & Libraries
- **Testing**: Jest, Vitest, Playwright
- **State Management**: Zustand, Redux Toolkit
- **API**: RESTful, GraphQL when needed
- **Authentication**: NextAuth.js, JWT

## Technologies to Avoid/Limit
- jQuery (unless maintaining legacy code)
- Large monolithic architectures
- Overly complex abstractions

## Decision Criteria
- Developer experience and productivity
- Community support and ecosystem
- Performance requirements
- Team expertise and learning curve
